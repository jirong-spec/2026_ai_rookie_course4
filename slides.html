為什麼模型不直接吃文字？

Lab 1 — 訊息結構轉換與一致性檢查
模型輸入不是純字串，而是 有格式的 Token 序列

你以為的輸入

# 直接把文字丟進去？
text = "用戶：我要退款 助理："

模型實際吃的格式

<|system|>
你是專業客服助理。<|end|>
<|user|>
我要退款<|end|>
<|assistant|>

apply_chat_template 的兩種用法

情境參數
訓練（完整對話）add_generation_prompt=False
推理（讓模型接著生）add_generation_prompt=True
⚠️ 這個差異貫穿整門課
搞錯 True/False，模型會在錯誤的地方停止或繼續生成。Lab6 的消融實驗就是為了量化這個錯誤有多嚴重。

Token 成本
訓練前先算錢

Lab 2 — Tokenizer 比較與訓練預算估算
同樣的中文，不同模型 Token 數差很多

為什麼 Token 數很重要？

  • 訓練費用 = token 數 × 單價
  • 超過 context window → 截斷
  • 選模型時的重要參考指標

訓練預算公式

total_tokens =
  筆數 × (prompt + response) × epochs

train_hours =
  total_tokens / tokens_per_sec / 3600

Lab 2 做的事

  • 對同一段文字，跑多個模型的 Tokenizer
  • 算 raw token（純文字）
  • 算 chat token（套 template 後）
  • 假設場景，呼叫估算函式
實務習慣：訓練開始前，先用這個公式估成本。10,000 筆 × 500 tokens × 3 epochs ÷ 15,000 tok/s ≈ 1,000 秒

04

LoRA / QLoRA
不改原始模型，只插旁邊

Lab 4 — SFT + PEFT 指令微調
LoRA 的核心概念

Full Fine-tuning（傳統）

更新所有參數 → 顯存需求極大，難以分發

# 1.1B 模型 → 約 4.4 GB（fp32）
# 加上梯度 → 需要 13GB+ 顯存

LoRA（Low-Rank Adaptation）

插入低秩矩陣 A、B，只訓練這兩個

W' = W + α × (B × A)
# W 凍結，只更新 B 和 A
# rank=16 → 可訓練參數大幅減少

QLoRA = LoRA + 4-bit 量化

  • Base model 以 4-bit（NF4）載入
  • 計算用 bfloat16
  • 同樣效果，顯存再省 50%+

本次訓練設定

rank (r)16
alpha32
Epochs5
Adapter 大小幾 MB（vs 模型 2GB+）
Lab4 關鍵程式碼
# 1. 設定 LoRA
lora_cfg = LoraConfig(
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
    target_modules=[
        "q_proj", "k_proj", "v_proj",
        "o_proj", "gate_proj",
        "up_proj", "down_proj",
    ],
)
peft_model = get_peft_model(base_model, lora_cfg)
# 2. 訓練
trainer = SFTTrainer(
    model=peft_model,
    train_dataset=ds_proc["train"],
    processing_class=tokenizer,
    args=SFTConfig(
        num_train_epochs=5,
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,
        learning_rate=1e-4,
        bf16=True,
    ),
)
trainer.train()
trainer.save_model("adapter")
target_modules：LoRA 插入的位置。涵蓋所有 attention 層（q/k/v/o_proj）和 FFN 層（gate/up/down_proj）可獲得最好效果。

六個帶走的 核心觀念

🔑 Chat Template 一致性

訓練和推理必須用相同 template。add_generation_prompt 一字之差，輸出天差地遠。

🔑 LoRA / QLoRA

不改原始權重，插入低秩矩陣。Adapter 幾 MB，4-bit 量化省顯存，是現在 fine-tuning 的標準做法。

🔑 SFT 資料格式

每筆資料是 messages 列表（role + content）。資料品質比資料量更重要。

🔑 Offline 評估

用規則或 LLM-as-judge 自動打分。不需人工標記就能快速迭代，再逐步升級為 BLEU / ROUGE。

🔑 消融實驗

隔離單一變數，量化它的影響。是工程決策驗證的最直接方式。

🔑 Token 成本意識

同樣的文字，不同 tokenizer 的 token 數可差 2~3 倍。訓練前先估算成本是基本工。
